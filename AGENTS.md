# Engineering Agent Guide

This repository may be worked on by multiple AI coding assistants.

## Before coding

-   Read the canonical documents in `docs/canon/`.
-   Follow the approved technology stack.
-   Respect repository boundaries.
-   Do not introduce new frameworks without approval.

## Priorities

1.  Product Philosophy
2.  Experience Canon
3.  Design Canon
4.  Engineering Canon

## Quality Standards

-   Type-safe code
-   Accessible UI
-   Reusable components
-   Clean architecture
-   Tests for critical functionality
-   Clear commit history

## Repository Layout

-   apps/ --- applications
-   packages/ --- shared libraries
-   assets/ --- design assets
-   docs/ --- documentation
-   supabase/ --- database resources

Every implementation should strengthen the canonical architecture rather
than bypass it.
