# Purpose

This document defines the permanent AI engineering standards for EZ. It
establishes how artificial intelligence is designed, integrated,
evaluated, and governed throughout the product.

AI is a capability that enhances the user's journey---it never replaces
the user's decisions.

------------------------------------------------------------------------

# AI Philosophy

EZ uses AI to reduce effort, increase confidence, and help users get
hired faster.

AI must always be:

-   Helpful
-   Transparent
-   Reliable
-   Secure
-   Explainable
-   Human-centered

------------------------------------------------------------------------

# Guiding Principles

-   AI assists rather than controls.
-   Users remain the final decision makers.
-   Recommendations should be understandable.
-   AI should reduce complexity, not create it.
-   Fail safely and recover gracefully.

------------------------------------------------------------------------

# AI Responsibilities

AI may:

-   Recommend jobs
-   Improve resumes
-   Draft cover letters
-   Prepare interview answers
-   Summarize information
-   Organize job-search activities
-   Surface insights and reminders

AI must never take irreversible actions without explicit user approval.

------------------------------------------------------------------------

# AI Architecture

    User
       ↓
    AI Interface
       ↓
    Prompt Layer
       ↓
    Context Builder
       ↓
    Model Provider
       ↓
    Tool Execution
       ↓
    Response Validation
       ↓
    User

Each layer should have a single responsibility.

------------------------------------------------------------------------

# Prompt Engineering

Prompts should:

-   Be modular
-   Be version controlled
-   Minimize ambiguity
-   Include only relevant context
-   Produce structured outputs whenever possible

Avoid embedding business rules directly into prompts when they belong in
application logic.

------------------------------------------------------------------------

# Context Management

Only provide the context necessary to complete the current task.

Context may include:

-   User profile
-   Resume
-   Active application
-   Interview history
-   User preferences

Exclude irrelevant information to improve quality and reduce cost.

------------------------------------------------------------------------

# Model Strategy

Engineering should support multiple AI providers through a common
abstraction.

Benefits include:

-   Reliability
-   Vendor flexibility
-   Cost optimization
-   Future extensibility

Application logic should not depend on a specific model.

------------------------------------------------------------------------

# Tool Calling

AI tools should be:

-   Explicit
-   Permission-aware
-   Auditable
-   Recoverable

Tool execution should validate inputs and outputs before affecting
application state.

------------------------------------------------------------------------

# Memory

Separate memory into:

-   Session memory
-   Conversation memory
-   Long-term user preferences

Memory must always respect user privacy and applicable retention
policies.

------------------------------------------------------------------------

# Safety

AI responses should:

-   Avoid fabrication where possible
-   Identify uncertainty
-   Reject harmful requests
-   Protect user privacy
-   Respect system permissions

------------------------------------------------------------------------

# Human-in-the-Loop

Require user confirmation before actions such as:

-   Sending emails
-   Submitting job applications
-   Modifying resumes
-   Sharing personal information

Recommendations may be automatic; actions require consent.

------------------------------------------------------------------------

# Failure Recovery

When AI cannot complete a task:

-   Explain what happened
-   Preserve user work
-   Offer alternatives
-   Allow retry without losing context

------------------------------------------------------------------------

# Performance & Cost

Optimize AI usage by:

-   Reusing context appropriately
-   Avoiding unnecessary requests
-   Selecting the appropriate model for each task
-   Monitoring latency and token usage

------------------------------------------------------------------------

# Observability

Track:

-   Success rate
-   Latency
-   Failure rate
-   User feedback
-   Prompt versions
-   Tool execution history

Use metrics to improve AI quality over time.

------------------------------------------------------------------------

# Testing

Validate:

-   Prompt behavior
-   Tool integrations
-   Safety constraints
-   Structured outputs
-   Regression scenarios

AI changes should be evaluated before release.

------------------------------------------------------------------------

# Governance

Before introducing a new AI capability, ask:

-   Does it solve a real user problem?
-   Is it transparent?
-   Is user consent required?
-   Can failures be recovered safely?
-   Does it align with the Product Philosophy, Experience Canon, and
    Design Canon?

If any answer is uncertain, redesign before implementation.
