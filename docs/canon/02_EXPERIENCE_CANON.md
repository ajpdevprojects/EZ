# EZ Experience Canon

## Purpose

The Experience Canon defines **how EZ should feel**, not how it should
look.

It is independent of UI, visual design, navigation, and implementation.

Every future workflow, screen, AI behavior, automation, notification,
and interaction must be measured against this document.

------------------------------------------------------------------------

# Core Experience Identity

## The Vehicle Principle

> **EZ is the vehicle.**
>
> **The user's career destination is the destination.**

EZ exists for one purpose:

**To carry users from needing a job to landing a job with the least
possible friction.**

People do not use EZ because they enjoy managing applications.

People use EZ because they want employment.

EZ is simply the vehicle that helps them get there.

------------------------------------------------------------------------

# Delivery Principle

The hiring journey is intentionally simple.

``` text
Need a Job
    ↓
Apply
    ↓
Wait
    ↓
Recruiter Responds
    ↓
Interview
    ↓
Offer
    ↓
Hired
```

EZ does **not** add steps to this journey.

EZ removes unnecessary friction between the steps that already exist.

The product optimizes the outcome---not the process.

------------------------------------------------------------------------

# Core Experience Principles

## Easy

Using EZ should feel effortless.

Nothing should feel unnecessarily complicated.

## No-Brainer

The next step should always be obvious.

The user should never wonder:

-   What should I do next?
-   Did I miss something?
-   Which application matters?
-   Where is that email?
-   Which resume should I use?

EZ removes unnecessary thinking.

## Zero Friction

Every interaction should move the user forward.

No unnecessary searching.

No unnecessary clicks.

No duplicate work.

No manual tracking.

------------------------------------------------------------------------

# Emotional Objective

EZ transforms job hunting from an exhausting search into an exciting
daily journey.

The user should:

-   Feel excited to open EZ.
-   Feel supported.
-   Never feel overwhelmed.
-   Never feel lost.
-   Always feel progress.

------------------------------------------------------------------------

# Experience Philosophy

The journey is the product.

Not the dashboard.

Not the pipeline.

Not the AI.

Everything exists to support the user's journey toward employment.

------------------------------------------------------------------------

# Daily Experience

Every morning begins with a Daily Briefing.

The briefing answers:

-   What happened?
-   What matters?
-   What should I do next?

The user should understand today's journey in under one minute.

------------------------------------------------------------------------

# Home Experience

The Home is not a dashboard.

It is today's starting point.

The user should immediately know what matters and what to do next.

------------------------------------------------------------------------

# Pipeline Experience

The pipeline should feel alive.

Applications automatically evolve as the journey progresses.

The user should never feel like a project manager moving cards.

EZ manages the journey.

The user makes decisions.

------------------------------------------------------------------------

# Recruiter Experience

Recruiter replies become actions.

Instead of raw emails, EZ explains:

-   What happened
-   Why it matters
-   What should happen next

------------------------------------------------------------------------

# AI Experience

EZ behaves like an intelligent journey companion.

It remembers context.

It prepares.

It assists.

The user always remains in control.

------------------------------------------------------------------------

# Notification Philosophy

Notifications are **Journey Notifications**.

They exist because the journey moved forward.

Not because the app wants attention.

Examples:

-   Morning Briefing
-   New opportunity found
-   Recruiter replied
-   Interview scheduled
-   Interview reminder
-   Offer received
-   Journey completed

------------------------------------------------------------------------

# Journey Sound Language

EZ does not use traditional notification sounds.

Users choose a **Journey Theme**.

Each Journey Theme contains a complete sonic language.

Themes:

-   🚗 Executive (Default)
-   🎹 Minimal
-   🌌 Ambient
-   🌿 Nature
-   🔕 Silent

Users do not assign random sounds.

Every milestone shares the same sonic identity.

------------------------------------------------------------------------

# Journey Archive

Every completed journey deserves to be remembered.

The Journey Archive preserves **the story**, not just the data.

## Timeline

Examples:

-   Journey Started
-   Resume Completed
-   Application Submitted
-   Recruiter Viewed
-   Recruiter Replied
-   Interview Scheduled
-   Interview Completed
-   Offer Received
-   Offer Accepted
-   Journey Completed

## Journey Summary

Each archive may preserve:

-   Journey start date
-   Journey completion date
-   Journey duration
-   Company hired by
-   Position accepted
-   Applications submitted
-   Recruiter conversations
-   Interviews completed
-   Offers received
-   Journey milestones
-   Personal reflections (optional)
-   Final outcome

Years later the user should be able to relive the complete experience.

------------------------------------------------------------------------

# Experience Validation

Every future feature must answer:

-   Does this make EZ easier?
-   Does this remove thinking?
-   Does this reduce friction?
-   Does this move the user closer to employment?
-   Does this support the Vehicle Principle?

If not, it should be reconsidered.
