# Purpose

This document defines the permanent component architecture for EZ. It
ensures every interface is built from reusable, composable, and
consistent building blocks that faithfully implement the Design Canon
and Experience Canon.

------------------------------------------------------------------------

# Component Philosophy

Components are products, not placeholders.

Every component should be:

-   Reusable
-   Predictable
-   Accessible
-   Performant
-   Easy to understand
-   Focused on a single responsibility

Build once, reuse everywhere.

------------------------------------------------------------------------

# Component Hierarchy

    Design Tokens
          ↓
    Primitive Components
          ↓
    Shared Components
          ↓
    Feature Components
          ↓
    Page Sections
          ↓
    Screens / Pages

Higher layers compose lower layers. Lower layers never depend on higher
ones.

------------------------------------------------------------------------

# Component Categories

## Design Tokens

Colors, typography, spacing, radius, shadows, motion, and icon sizing.

## Primitive Components

The smallest UI building blocks.

Examples:

-   Button
-   Input
-   Text
-   Icon
-   Badge
-   Divider
-   Avatar

## Shared Components

Reusable combinations of primitives.

Examples:

-   SearchBar
-   EmptyState
-   Modal
-   Card
-   Tabs
-   Toast
-   NavigationBar

## Feature Components

Components that belong to one feature.

Examples:

-   JobCard
-   InterviewTimeline
-   ResumeUploader
-   CareerProgressChart

## Screens

Screens assemble feature components into complete user experiences.

------------------------------------------------------------------------

# Single Responsibility

Each component should solve one problem.

If a component has multiple unrelated responsibilities, split it.

------------------------------------------------------------------------

# Composition Over Inheritance

Prefer composing small components into larger experiences.

Avoid large monolithic components.

------------------------------------------------------------------------

# Props Standards

Props should be:

-   Explicit
-   Strongly typed
-   Minimal
-   Predictable

Avoid boolean explosions and unnecessary configuration.

------------------------------------------------------------------------

# State Boundaries

A component should own only the state it truly needs.

Shared state belongs in feature logic or application state---not deeply
nested UI components.

------------------------------------------------------------------------

# Styling Rules

All styling must come from the Design Canon.

Never introduce one-off visual styles that conflict with the design
system.

------------------------------------------------------------------------

# Accessibility

Every interactive component must support:

-   Keyboard navigation
-   Focus indicators
-   Screen readers
-   Proper semantic markup

Accessibility is mandatory.

------------------------------------------------------------------------

# Performance

Components should:

-   Avoid unnecessary re-renders
-   Load only required data
-   Keep rendering predictable
-   Be optimized only when profiling identifies a real need

------------------------------------------------------------------------

# Folder Organization

A recommended structure:

    components/
        primitives/
        shared/
    features/
        jobs/
        applications/
        interviews/

Keep feature-specific components inside their feature until they become
broadly reusable.

------------------------------------------------------------------------

# Documentation

Shared components should document:

-   Purpose
-   Props
-   Usage
-   Accessibility considerations
-   Examples

------------------------------------------------------------------------

# Governance

Before creating a new component, ask:

-   Does one already exist?
-   Can an existing component be extended?
-   Will this improve consistency across EZ?

If the answer is no, redesign before implementation.
